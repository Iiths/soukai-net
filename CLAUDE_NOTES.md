## CoWork VM 固有の制限事項（2026-04-25 確認）

| 制限 | 内容 | 回避策 |
|------|------|--------|
| `.git` ディレクトリへの書き込み不可 | `git add/commit` が index 破損を起こす場合がある。index.lock も削除不可 | IS様にPowerShellで手動git操作をお願いする。またはClaude Codeに移行する |
| bash タイムアウト 45秒 | スクレイピング等の長時間処理が途中停止する | Phase分割＋エピソードごとのキャッシュ保存で中断・再開可能にする |
| `python3 script.py` の不安定な起動 | 長いスクリプトを直接実行するとプロセスが立ち上がらない場合がある | `importlib.util.spec_from_file_location` 経由でモジュールとして読み込む |
| Write/bash heredoc でのファイル切れ | 長いファイル（300行超）を1度に書くと末尾が切れることがある | 書き込み後に `wc -l` で行数確認 → 不足分はEditで追記 |

## ファイル書き込み後の必須チェック手順（2026-05-09 制定）

`.ts` / `.tsx` ファイルを新規作成・上書き・追記した後は、**必ず以下の2ステップを実行すること**。

### Step 1 — 末尾確認（案B）

```bash
wc -l <ファイルパス>   # 期待行数と一致するか確認
tail -5 <ファイルパス>  # 末尾が正しく閉じているか目視確認
```

- 関数・クラス・コンポーネントの閉じ括弧 `}` や `export default` が末尾にあることを確認する
- 行数が明らかに少ない場合は切れているので、Editで不足分を追記する

### Step 2 — TypeScriptコンパイルチェック（案A）

```bash
cd /sessions/.../mnt/CloudeCode/soukai-net
npx tsc --noEmit 2>&1 | grep -v "npm notice"
```

- 出力が空（exit 0）になればOK
- エラーが出た場合はその場で修正してから次の作業に進む

### 適用タイミング

| 操作 | チェック要否 |
|------|------------|
| `.ts` / `.tsx` の新規作成 | **必須** |
| `.ts` / `.tsx` の編集（Editツール） | **必須** |
| `.js` / `.css` / `.json` の編集 | 任意（末尾確認のみ推奨） |
| データファイル（`*.json`）の更新 | 任意 |
